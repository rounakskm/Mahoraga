# Phase 2 Redesign — real-data, proven-libs, leaner fortress

**Status:** Proposed 2026-06-22 (supersedes parts of [`spec.md`](spec.md), [`plan.md`](plan.md), [`tasks.md`](tasks.md))
**Predecessor:** P2.1 wall framework (merged, PR #40)
**Driver:** operator decisions 2026-06-21/22 (real-data-only, proven stat libs, latest deps, fundamentals-later)

---

## 1. Why this revision

The original five-wall plan assumed synthetic-data generation, hand-rolled statistics, and a cross-sectional universe. Four operator decisions change that:

1. **Real data only — no synthetic.** Backtest on actual SPY history. (P2.8 synthetic-data is **dropped**.)
2. **Proven stat libraries, not hand-rolled.** Use **RiskLabAI** (BSD-3 — DSR/PSR/PBO/CPCV, validated mathematically correct) + **quantstats** (Apache-2.0 — tearsheets/metrics). Avoid mlfinlab (proprietary) and pypbo (AGPL).
3. **Latest pandas/numpy** (3.0.3 / 2.4.6) — greenfield, no legacy pin. Phase-1 suite verified green on it.
4. **Goal = trader + investor, year-over-year profit.** Fundamentals will narrow the universe later — but as a **selection layer upstream of the walls, not a wall** (deferred; separate design).

## 2. What a "wall" is (and isn't)

A wall is a **validation gate** — it answers *"is this strategy's edge real, or an artifact of overfitting / luck / fragility?"* It does **not** generate alpha and is **not** a stock screen. Strategies are **learned by the Phase-3 autoresearch loop**; the walls are the loop's fitness predicate, which is exactly why they must exist before any strategy is learned.

## 3. Revised shape — 4 walls + 3 gates (was 5 walls + synthetic-data)

| Wall | Question | How (libraries) |
|---|---|---|
| **W1 — Statistical Rigor** | Is the edge statistically real, not multiple-testing luck? | **RiskLabAI**: PBO (the headline overfitting metric), DSR/PSR; quantstats for Sharpe/Sortino/CVaR/drawdown |
| **W2 — Complexity Control** | Is it robust to its own parameters, not curve-fit? | parameter-sensitivity perturbation (±10/20%), rolling-window stability, MDL penalty for param count (reuses the Phase-1 backtest engine) |
| **W3 — Generalization** | Does it hold out-of-sample, not just in-sample? | **walk-forward / OOS** on the SPY history + multi-regime split (Phase-1 regime detector). *(cross-asset rotation deferred until >1 instrument)* |
| **W4 — Meta-Awareness** | Are we fooling ourselves across many trials? | trial-budget tracker (multiple-comparison count feeding W1's PBO/DSR #-trials); KB forbidden-pattern stub (Hindsight wiring lands Phase 3) |
| **Gates** | Aggregate pass/fail | Fitness / Robustness / Risk gates over the WallReports; AND aggregation |

**Folded in:** the original "Wall 2 Data Discipline" (combinatorial purged CV) is **absorbed into W1** — RiskLabAI's CPCV *is* the data-discipline core; we keep its PIT-eval leak check as a W1 sub-test rather than a separate wall.

## 4. How the walls are validated (no hand-picked strategy needed)

A statistical gate is tested with **controlled return-series fixtures with known ground truth**, the same way the RiskLabAI validation worked:

- **pure-noise returns → walls REJECT** (PBO ≈ 0.5, DSR low)
- **injected persistent-edge returns → walls PASS**

This proves wall *correctness* without picking any trading strategy.

## 5. The one real-data end-to-end smoke (calibration)

To prove the walls work through the *full pipeline* on *real SPY data* (not just unit fixtures), one published strategy as a **throwaway integration fixture** — explicitly NOT "Mahoraga's strategy" (those are learned in Phase 3):

- **Known-good — Faber 200-day SMA timing on SPY.** Hold SPY when price > 200-day SMA, else flat. One parameter. The most-replicated simple single-asset tactical rule (Faber 2007, *A Quantitative Approach to Tactical Asset Allocation*), documented to survive out-of-sample across decades. The walls must **PASS** it.
- **Known-bad — deliberate overfit canary.** A multi-parameter SMA/threshold rule grid-searched to fit one window (e.g. the 2020 COVID crash): high in-sample Sharpe, no OOS basis. The walls must **REJECT** it — **PBO is the discriminator**.

**Phase-2 exit proof (CI gate):** on real SPY data, walls + gates promote Faber-SMA and reject the overfit canary.

## 6. Data

SPY ~10yr daily already loaded via the Phase-1 foundation: 2,882 bars, 2015-01-02 → 2026-06-18, 0 NYSE-calendar gaps, at `data/parquet/ohlcv/SPY/` (`scripts/pull_spy_daily.py`, idempotent). Raw + adjusted close + PIT fields present.

## 7. Dependency stack

Add: `RiskLabAI` (BSD-3), `quantstats` (Apache-2.0). Already on pandas 3.0.3 / numpy 2.4.6.

**RiskLabAI wrapper (required — genuine correctness caveats from validation):**
- assemble DSR from `benchmark_sharpe_ratio` (E[max SR]) + `probabilistic_sharpe_ratio` (no one-call DSR);
- feed an **effective-independent-trials** count (it uses raw N → anti-conservative DSR for correlated strategies);
- **NaN-guard PBO inputs** (NaN propagates silently to a wrong number).
